Java阻塞队列实现原理与使用场景（生产者-消费者模式）记录

一、什么是阻塞队列

阻塞队列常用于消费者-生产者模型，阻塞队列是生产者与消费者对元素进行操作的容器，生产者线程往队列中添加数据，消费者线程从队列中取出数据。Java中提供了7种阻塞队列，分别为ArrayBlockingQueue(由数组结构组成的有界阻塞队列)，LinkedBlockingQueue(由链表结构组成的有界阻塞队列)，PriorityBlockingQueue(支持优先级排序的无界阻塞队列)，DelayQueue(使用优先级队列实现的无界阻塞队列)，SynchronousQueue(不存储元素的阻塞队列)，LinkedTransferQueue(由链表结构组成的无界阻塞队列)，LinkedBlockingDeque(由链表结构组成的双向阻塞队列)。

二、阻塞队列的阻塞场景

1.当队列中没有元素的情况下，消费者线程会被自动阻塞，直到有元素放入队列

2.当队列中满元素的情况下，生产者线程会被自动阻塞，直到有元素被取出

三、ArrayBlockingQueue的实现原理

ArrayBlockingQueue.java

ArrayBlockingQueue类的几个主要属性含义如下：

属性 含义
Object[] items Object类型数组，存放队列元素
int count 队列中元素个数
int takeIndex 队首元素的下标
int putIndex 队尾元素的下标
ReentrantLock lock 可重入锁，用于多线程对队列的同步操作
Condition notEmpty 条件对象，队列空时，让消费者线程阻塞，生产者线程存入元素时，解除消费者线程的阻塞
Condition notFull 条件对象，队列满时，让生产者线程阻塞，消费者线程取出元素时，解除生产者线程的阻塞

从ArrayBlockingQueue源码中不难看出，当生产者线程要往队列的队尾插入元素时，会调用put方法，put方法的具体实现中，首先初始化可重入锁ReentrantLock对象，并调用lockInterruptibly方法获得可中断锁，然后判断队列是否满元素，如果元素已满的情况下，是不能继续执行插入操作的，因此使用条件对象notFull的await方法，让当前生产者线程进入等待状态，当该线程被唤醒时（notFull.signalAll()被调用后），执行enqueue方法让元素入队列，enqueue方法中，首先将items数组的队尾元素赋值为x，并执行putIndex++和count++操作，执行元素入队，入队完成后，调用条件对象notEmpty的signalAll方法唤醒所有等待的消费者线程，这样消费者线程便可以从队列中执行取元素的操作。

与put方法类似，take方法在消费者线程从队列中取元素时被调用，首先获得可中断重入锁，然后判断队列是否为空，如果为空，则调用条件对象notEmpty的await方法，将当前消费者线程挂起进入等待状态，直到生产者插入元素成功，notEmpty的signalAll方法被调用，执行dequeue方法取出队首元素，并调用notFull的signAll，唤醒所有等待的生产者线程。

从以上源码阅读很容易看出，ArrayBlockingQueue元素入队和出队的实现原理与操作系统同步概念中的提到的消费者-生产者模型类似，而其中的条件对象则相当于操作系统中的信号量。
